# DMPO Minimal Runbook

This note documents the current repository layout and the smallest reliable commands for
Robomimic image-based MeanFlow pretraining and evaluation.

## What This Repository Does

DMPO is a robot policy training codebase built around conditional generative policies.
The main workflow is:

1. Stage 1 pretraining: learn a policy from demonstration data.
2. Stage 2 finetuning: improve the pretrained policy with online RL, mainly PPO.
3. Evaluation: roll out checkpoints in simulator environments and report success rate,
   reward, trajectory length, and inference speed.

The Robomimic image setting uses observations with:

- `rgb`: image observation, e.g. wrist camera image.
- `state`: proprioceptive low-dimensional robot state.
- `action`: action chunks with horizon length `horizon_steps`.

For image MeanFlow, the model is not only a vision encoder. It trains the whole policy:

- ViT image encoder: `model/common/vit.py`
- image/state fusion and MeanFlow velocity network: `model/flow/mlp_meanflow.py`
- MeanFlow loss: `model/flow/shortcut_dispersive.py`
- pretraining loop: `agent/pretrain/train_agent.py`
- image eval loop: `agent/eval/eval_meanflow_img_agent.py`

## Directory Map

Core entrypoint:

```bash
script/run.py
```

Configs:

```bash
cfg/robomimic/pretrain/<task>    # image/state BC-style pretraining configs
cfg/robomimic/eval/<task>        # rollout/evaluation configs
cfg/robomimic/finetune/<task>    # PPO/RL finetuning configs
cfg/robomimic/env_meta           # Robomimic env metadata JSON files
cfg/gym                          # D4RL / MuJoCo state-based tasks
cfg/furniture, cfg/d3il          # additional task suites
```

Training agents:

```bash
agent/pretrain   # offline pretraining loops
agent/finetune   # online RL finetuning loops
agent/eval       # rollout evaluation loops
agent/dataset    # npz sequence datasets
```

Models:

```bash
model/flow       # MeanFlow, ReFlow, Shortcut, Consistency models
model/diffusion  # diffusion-policy variants
model/gaussian   # Gaussian/GMM baselines and PPO variants
model/loss       # dispersive loss
model/common     # MLP, ViT, shared modules
```

Environment wrappers:

```bash
env/gym_utils
```

Data processing:

```bash
data_process
script/dataset
```

Outputs:

```bash
data/robomimic/<task>-img       # train.npz and normalization.npz
log/robomimic/pretrain/...      # checkpoints and training logs
log/robomimic/eval/...          # eval outputs
visualize/...                   # eval plots/statistics generated by eval_agent_base.py
wandb_offline/...               # offline W&B runs
```

## Minimal Stage 1 Pretraining

Available Robomimic image tasks:

```bash
lift
can
square

```

### Recommended First Sanity Run: No Dispersive Loss

This checks whether the base MeanFlow BC pipeline can learn before adding dispersive
regularization.

```bash
TASK=lift

python script/run.py \
  --config-dir=cfg/robomimic/pretrain/${TASK} \
  --config-name=pre_meanflow_dispersive_mlp_img \
  model.dispersive_loss_weight=0.0 \
  model.max_denoising_steps=1 \
  wandb.offline_mode=true
```

Checkpoints will be written under:

```bash
log/robomimic/pretrain/${TASK}/...
```

### Dispersive MeanFlow Run

Start with a small dispersive weight. In the current implementation, `infonce_l2`
can make the total loss negative, so `0.5` may dominate the MeanFlow MSE objective.

```bash
TASK=lift

python script/run.py \
  --config-dir=cfg/robomimic/pretrain/${TASK} \
  --config-name=pre_meanflow_dispersive_mlp_img \
  model.max_denoising_steps=1 \
  model.dispersive_loss_type=infonce_l2 \
  model.dispersive_loss_weight=0.01 \
  wandb.offline_mode=true
```

Alternative non-negative dispersive loss:

```bash
TASK=lift

python script/run.py \
  --config-dir=cfg/robomimic/pretrain/${TASK} \
  --config-name=pre_meanflow_dispersive_mlp_img \
  model.max_denoising_steps=1 \
  model.dispersive_loss_type=hinge \
  model.dispersive_loss_weight=0.01 \
  wandb.offline_mode=true
```

## Minimal Eval

Use `base_policy_path`, not `checkpoint_path`.

For Robomimic image eval, add `+sim_device=0` so the code uses EGL on GPU 0
instead of falling back to `osmesa`.

Start with small parallelism:

```bash
TASK=lift
CKPT=/absolute/path/to/state_2000.pt

python script/run.py \
  --config-dir=cfg/robomimic/eval/${TASK} \
  --config-name=eval_meanflow_mlp_img \
  base_policy_path=${CKPT} \
  denoising_step_list='[1]' \
  denoising_steps=1 \
  +sim_device=0 \
  env.n_envs=5
```

For a quick smoke test:

```bash
TASK=lift
CKPT=/absolute/path/to/state_2000.pt

python script/run.py \
  --config-dir=cfg/robomimic/eval/${TASK} \
  --config-name=eval_meanflow_mlp_img \
  base_policy_path=${CKPT} \
  denoising_step_list='[1]' \
  denoising_steps=1 \
  +sim_device=0 \
  env.n_envs=1 \
  n_steps=20
```

For fuller evaluation:

```bash
TASK=lift
CKPT=/absolute/path/to/state_2000.pt

python script/run.py \
  --config-dir=cfg/robomimic/eval/${TASK} \
  --config-name=eval_meanflow_mlp_img \
  base_policy_path=${CKPT} \
  denoising_step_list='[1]' \
  denoising_steps=1 \
  +sim_device=0 \
  env.n_envs=10 \
  n_steps=300
```

Increase `env.n_envs` carefully. Default `50` can crash Robomimic image rendering
workers on this machine.

## Reading Eval Results

Eval writes statistics to:

```bash
visualize/MeanFlow/<task>/<time>/eval_statistics.npz
visualize/MeanFlow/<task>/<time>/denoise_step.png
```

Print recent eval results:

```bash
python -c "import numpy as np, glob; \
paths=sorted(glob.glob('visualize/MeanFlow/lift/*/eval_statistics.npz'))[-5:]; \
for p in paths: \
    print('\\n'+p); \
    d=np.load(p)['data']; \
    [print({k: (row[k].item() if hasattr(row[k], 'item') else row[k]) for k in d.dtype.names}) for row in d]"
```

Interpretation:

- `num_episodes_finished=0`: eval did not run long enough to finish episodes.
- `num_episodes_finished>0` and `success_rate=0`: episodes finished, but policy failed.
- `avg_episode_reward=0` and `avg_best_reward=0`: in Lift, the object was never lifted.

## Common Errors

### Missing config

If Hydra says:

```text
Cannot find primary config 'eval_meanflow_mlp'
```

Use:

```bash
--config-name=eval_meanflow_mlp_img
```

### Checkpoint not loaded

Use:

```bash
base_policy_path=/path/to/state_xxx.pt
```

not:

```bash
checkpoint_path=/path/to/state_xxx.pt
```

### Robomimic worker EOFError

Usually a subprocess crashed during rendering or stepping. Use EGL and lower env count:

```bash
+sim_device=0 env.n_envs=1
```

Then increase:

```bash
env.n_envs=5
env.n_envs=10
```

### OpenGL_accelerate warning

This is not fatal:

```text
No OpenGL_accelerate module loaded
```

Optional:

```bash
pip install PyOpenGL_accelerate
```

### Success rate is all zero

First check whether episodes finished:

```text
num_episodes_finished
```

If episodes finished and reward is still zero, the policy did not solve the task.
In this checkout, a large `model.dispersive_loss_weight=0.5` with `infonce_l2`
can dominate the MeanFlow MSE loss and produce negative total losses, so first
validate `model.dispersive_loss_weight=0.0` or a much smaller value such as `0.01`.
